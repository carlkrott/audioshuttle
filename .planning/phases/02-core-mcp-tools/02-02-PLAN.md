  <action>
    **osc_bridge.py — New bridge methods:**

    Add these methods to `ReaperOSC` (all go through `send_command` which now validates addresses):

    1. `set_fx_param(track: int, fx: int, param: int, value: float) -> CommandResult`
       - Validate: track >= 1, fx >= 0, param >= 0, value 0.0-1.0 (clamp)
       - Send: `/track/{track}/fx/{fx}/fxparam/{param}/value` with float value
       - Reaper OSC FX indices are 0-based (fx 0 = first plugin on track)

    2. `fx_bypass(track: int, fx: int, bypass: bool) -> CommandResult`
       - Validate: track >= 1, fx >= 0
       - Send: `/track/{track}/fx/{fx}/bypass` with 1 (bypassed) or 0 (enabled)
       - NOTE: Reaper OSC convention: 1 = bypassed, 0 = active (inverted from what you'd expect)

    3. `trigger_action(command_id: int) -> CommandResult`
       - Validate: command_id > 0 (Reaper action IDs are positive integers)
       - Send: `/action` with integer command_id
       - This is the escape hatch — any Reaper action can be triggered

    4. `set_track_recarm(track: int, arm: bool) -> CommandResult`
       - Validate: track >= 1
       - Send: `/track/{track}/recarm` with 1 (armed) or 0 (disarmed)

    5. `toggle_repeat() -> CommandResult`
       - Send: `/repeat` with no args (toggles repeat on/off)
       - Actually send `/repeat` — Reaper toggles on any message

    6. `toggle_metronome() -> CommandResult`
       - Send: `/click` with no args (toggles metronome)
       - Same toggle pattern as repeat

    **models.py — Add FXState (optional, for future state tracking):**

    Add a lightweight model:
    ```python
    class FXState(BaseModel):
        """State of an FX plugin on a track."""
        track_number: int = Field(ge=1)
        fx_index: int = Field(ge=0)
        name: str = ""
        bypassed: bool = False
        params: dict[int, float] = Field(default_factory=dict)
        model_config = {"populate_by_name": True}
    ```

    Do NOT add FXState to DAWState yet — Reaper doesn't proactively send FX state, and polling all FX params would be too chatty. Just define the model for future use.

    Add 6 new MCP tools to `create_server()` in server.py:

    1. `set_fx_param(track: int, fx: int, param: int, value: float) -> dict[str, Any]`
       - Docstring: "Set an FX plugin parameter value. FX and param indices are 0-based."
       - Args descriptions in signature:
         - `track: Track number (starts at 1)`
         - `fx: FX plugin index on the track (starts at 0)`
         - `param: Parameter index within the FX (starts at 0)`
         - `value: Parameter value from 0.0 to 1.0`
       - Calls `bridge.set_fx_param(track, fx, param, value)`
       - Returns: `{"success": true, "track": 1, "fx": 0, "param": 2, "value": 0.5}`

    2. `fx_bypass(track: int, fx: int, bypass: bool) -> dict[str, Any]`
       - Docstring: "Bypass or enable an FX plugin on a track."
       - Args: `track: Track number (starts at 1)`, `fx: FX index (starts at 0)`, `bypass: True to bypass, False to enable`
       - Calls `bridge.fx_bypass(track, fx, bypass)`
       - Returns: `{"success": true, "track": 1, "fx": 0, "bypassed": true}`

    3. `trigger_action(command_id: int) -> dict[str, Any]`
       - Docstring: "Trigger a Reaper action by its command ID. Use this for any Reaper action not covered by other tools."
       - Args: `command_id: Reaper action command ID (positive integer, e.g. 40025 for 'Go to marker 1')`
       - Validates command_id > 0
       - Calls `bridge.trigger_action(command_id)`
       - Returns: `{"success": true, "action_id": 40025}`

    4. `set_track_arm(track: int, arm: bool) -> dict[str, Any]`
       - Docstring: "Arm or disarm a track for recording."
       - Args: `track: Track number (starts at 1)`, `arm: True to arm for recording, False to disarm`
       - Calls `bridge.set_track_recarm(track, arm)`
       - Returns: `{"success": true, "track": 1, "armed": true}`

    5. `toggle_repeat() -> dict[str, Any]`
       - Docstring: "Toggle repeat on/off in Reaper."
       - Calls `bridge.toggle_repeat()`
       - Returns: `{"success": true, "toggled": "repeat"}`

    6. `toggle_metronome() -> dict[str, Any]`
       - Docstring: "Toggle the metronome/click on/off in Reaper."
       - Calls `bridge.toggle_metronome()`
       - Returns: `{"success": true, "toggled": "metronome"}`
