  <name>Task 1: Add address validation + new bridge methods</name>
  <files>src/audioshuttle/osc_bridge.py, src/audioshuttle/models.py, tests/test_osc_bridge.py</files>
  <action>
    **osc_bridge.py — Address validation:**

    Add a `_validate_address` method and call it from `send_command` BEFORE sending. Validation rules:
    - Address must start with `/`
    - Address must match a known Reaper OSC pattern (whitelist approach)
    - No path traversal (`..`), no null bytes, no control characters
    - Track/FX numbers in addresses must be positive integers

    Known patterns whitelist (regex):
    ```
    /play, /stop, /record, /pause, /rewind, /forward
    /time
    /track/{n}/volume, /track/{n}/mute, /track/{n}/solo, /track/{n}/pan
    /track/{n}/name, /track/{n}/select, /track/{n}/recarm
    /track/{n}/fx/{n}/fxparam/{n}/value, /track/{n}/fx/{n}/bypass
    /track/count
    /master/volume, /master/pan
    /action
    /repeat, /click
    /marker/{n}/name, /marker/{n}/time
    /marker/count
    /track/{n}/send/{n}/volume
    ```

    If address doesn't match, return `CommandResult(success=False, address=address, error=f"Invalid OSC address: {address}")`. Log a warning. Do NOT send.

    **osc_bridge.py — New bridge methods:**

    Add these methods to `ReaperOSC`:

    1. `transport_seek(seconds: float) -> CommandResult`
       - Validates seconds >= 0
       - Sends `/time` with float value (Reaper interprets this as position in seconds)
       - Also update `self._state.transport.position_seconds` optimistically

    2. `get_track_count_real(self) -> int`
       - Sends `/track/count` and waits briefly for feedback
       - Falls back to the existing heuristic if no feedback received
       - Add `_track_count: int = 0` to internal state
       - Parse feedback: when address == `/track/count`, set `self._track_count`

    3. `set_master_volume(value: float) -> CommandResult` — already exists as a one-liner, verify it's there. If not, add it. Clamps 0.0-1.0.

    4. `set_master_pan(pan: float) -> CommandResult`
       - Clamps -1.0 to 1.0
       - Sends `/master/pan` with float value

    **osc_bridge.py — Feedback parsing updates:**

    In `_update_state`, add handlers for:
    - `/track/count` → store in `self._track_count`
    - `/master/volume` → store in `self._state.master_volume`
    - `/master/pan` → store in `self._state.master_pan`

    Note: Master feedback addresses are `/master/volume` and `/master/pan` (flat, no sub-parts like track volume).

    **models.py — Extended DAWState:**

    Add to `DAWState`:
    ```python
    track_count: int = 0
    master_volume: float = Field(default=0.75, ge=0.0, le=1.0)
    master_pan: float = Field(default=0.0, ge=-1.0, le=1.0)
    ```

    **tests/test_osc_bridge.py — New test class:**

    Add `TestAddressValidation` class:
    - `test_valid_transport_address` — `/play` passes
    - `test_valid_track_volume_address` — `/track/1/volume` passes
    - `test_rejects_no_leading_slash` — `play` rejected
    - `test_rejects_path_traversal` — `/track/../etc/passwd` rejected
    - `test_rejects_null_byte` — `/track/1\x00/volume` rejected
    - `test_rejects_unknown_pattern` — `/some/random/address` rejected
    - `test_rejects_negative_track` — `/track/-1/volume` rejected
    - `test_valid_fx_param_address` — `/track/1/fx/2/fxparam/3/value` passes

    Add `TestExtendedBridge` class:
    - `test_transport_seek_negative_rejected` — negative seconds returns error
    - `test_transport_seek_sends_correct_address` — mock the client, verify `/time` sent with float
    - `test_master_pan_clamped` — pan 2.0 clamped to 1.0

    IMPORTANT: Tests must NOT require Reaper running. Use the offline pattern from existing tests (instantiate bridge with unused ports, mock the UDP client if needed). For address validation tests, call `_validate_address` directly.

       - Docstring: "Set the master track volume."
       - Clamps volume to [0.0, 1.0]

       - Returns: `{"success": true, "volume": 0.75}`

    4. `set_master_pan(pan: float) -> dict[str, Any]`
       - Docstring: "Set the master track pan position."
       - Clamps pan to [-1.0, 1.0]
