  <action>
Create three utility modules and add new config fields:

**error_log.py** — Thread-safe error log:
- `ErrorLog` class with `max_entries: int = 200` default
- `add(message: str, level: str = "error")` — appends with ISO timestamp, trims to max_entries
- `get_recent(n: int = 50) -> list[dict]` — returns last N entries as `{"timestamp": str, "level": str, "message": str}`
- `clear()` — empties the log
- Use `threading.Lock` for thread safety
- Module-level `error_log = ErrorLog()` singleton

**gpu_monitor.py** — AMD GPU VRAM from sysfs:
- `get_gpu_vram(card_index: int = 1) -> dict` — reads `/sys/class/drm/card{card_index}/device/mem_info_vram_total` and `mem_info_vram_used`
- Returns `{"vram_total_mb": int, "vram_used_mb": int, "vram_used_pct": float}`
- On FileNotFoundError (no GPU), returns zeros
- On this system: card0 = iGPU, card1 = RX 6950 XT (16GB)

**daw_detect.py** — DAW process detection:
- `detect_daw() -> dict[str, Any]` — returns `{"reaper": {"running": bool}, "ardour": {"running": bool}, "detected": str}` where `detected` is the name of the first running DAW or "none"
- Uses `subprocess.run(["pgrep", "-c", "-x", pattern], capture_output=True, text=True, timeout=2)` for each DAW
- IMPORTANT: always use `pgrep -x` (exact match) — without `-x`, `pgrep reaper` matches `oom_reaper` kernel thread
- `detected` field: returns "reaper" if Reaper running, "ardour" if Ardour running, "none" otherwise (prefer Reaper if both running)

**config.py** — Add new fields to Settings class:
- `daw_type: str = "reaper"` — "reaper" or "ardour"
- `auto_open_browser: bool = True` — auto-open Chromium on startup
- `tray_enabled: bool = True` — show system tray icon
- `toast_notifications: bool = True` — show system tray toasts for errors
- `log_level: str = "warning"` — logging level for uvicorn

Do NOT modify any existing fields. Only add new ones.

- Stores shared state on `app.state`: settings, templates, bridge, model_server, translator, error_log (from error_log module)

**web_routes/__init__.py** — Empty init file

**web_routes/home.py** — Home/Status route:
- `router = APIRouter()`
- `GET /` — renders home.html with:
  - `status` dict containing:
    - `daw_connected: bool` (from bridge.is_connected if bridge exists, else False)
    - `daw_name: str` (from settings.daw_type.title())
    - `mcp_running: bool` (always True if web server is responding)
    - `model_loaded: bool` (from model_server.is_running if model_server exists, else False)
    - `model_state: str` — "running" if model_server.is_running, "loading" if model_server exists but not running, "disabled" if no model_server
    - `gpu_vram: dict` (from gpu_monitor.get_gpu_vram(card_index=1))
  - `errors: list` (from error_log.get_recent(50))
- IMPORTANT: All bridge/model_server access must be None-safe — check `if bridge is not None` before accessing. The web app may be created without a running bridge.

**templates/base.html** — Shared layout:
- HTML5 document with `<head>` containing: title "AudioShuttle", viewport meta, inline `<style>` block (no external CSS)
- Inline styles — dark theme (background #1a1a2e, text #eee, accent #0f9b58 green for status OK, #ea4335 red for errors, #fbbc04 yellow for warnings)
- `<nav>` with tab links: Home (/), Input (/input), Output (/output). Active tab highlighted via `request.path` comparison in Jinja2
- Status bar at bottom of nav or top of page showing DAW connected indicator
- `{% block content %}{% endblock %}` for page content
- Keep CSS minimal — this is a configurator, not a consumer app. Functional > pretty.

**templates/home.html** — Extends base.html:
- Status badges section: horizontal row of badge/pill elements showing each status indicator from the status dict. Use colored spans: green=connected/running, red=disconnected/error, yellow=loading

- If no errors, show "No errors logged" message
- Do NOT add any JavaScript. Pure server-rendered HTML. Page refresh to update.
