Create the system tray icon module:

```python
import pystray
from PIL import Image, ImageDraw
import webbrowser
import threading
from typing import Callable, Optional
```

**class TrayIcon:**
- `__init__(self, web_url: str = "http://127.0.0.1:8765", on_quit: Callable | None = None)`
- Stores web_url and on_quit callback
- Creates pystray.Icon with:
  - Name: "AudioShuttle"
  - Icon image: generated via `_create_icon_image()` (see below)
  - Menu: "Open Web UI" (default action), separator, "Quit"

**Methods:**
- `_create_icon_image() -> Image` — Generates a 64x64 icon programmatically using Pillow ImageDraw. Simple design: dark background (#1a1a2e), green (#0f9b58) rounded rectangle representing a shuttle/arrow shape. Keep it simple — a green rounded rectangle is fine.

- `start() -> None` — Calls `self.icon.run()` — BLOCKING, must be called from main thread

- `stop() -> None` — Calls `self.icon.stop()` — can be called from any thread

- `notify(message: str, title: str = "AudioShuttle") -> None` — Calls `self.icon.notify(message, title)` if `Icon.HAS_NOTIFICATION` is True, otherwise silently ignored

- `_on_open_web(icon, item)` — Opens `webbrowser.open(self.web_url)`

- `_on_quit(icon, item)` — Calls `self.icon.stop()` then `self.on_quit()` if provided

**Module-level convenience:**
- `create_icon(web_url: str, on_quit: Callable | None = None) -> TrayIcon` — factory function

**CRITICAL CONSTRAINTS:**
- `icon.run()` is BLOCKING and MUST run in the main thread (macOS requirement, good practice on Linux too)
- `icon.stop()` can be called from any thread (including signal handlers)
- On Linux with appindicator backend: menus work, notifications work
- On Xorg without desktop env: menus NOT supported, only default action
- Handle ImportError for pystray/Pillow gracefully — if not installed, TrayIcon.start() should log a warning and NOT crash

**Error handling:**
- If pystray or Pillow not installed (tray optional deps), TrayIcon should still be creatable but `start()` should log "pystray not installed, tray icon disabled" and return immediately
- Use a try/except ImportError at module level to set a `_TRAY_AVAILABLE = False` flag

Create the unified launcher and update CLI entry point:

**launcher.py** — Orchestration layer:

```python
def launch(
    settings: Settings,
    *,
    transport: str = "stdio",
    no_browser: bool = False,
    no_tray: bool = False,
    host: str = "127.0.0.1",
    port: int = 8765,
) -> None:
```

**Startup sequence (order matters):**

1. **Configure logging** — `logging.basicConfig(level=getattr(logging, settings.log_level.upper()))`

2. **Create MCP server** — call `create_server(settings)` which starts model server internally

3. **Create web app** — call `create_web_app(settings, bridge=..., model_server=..., translator=...)` — BUT we need the bridge/model_server from create_server. Solution: create the bridge and model_server separately, pass to both create_server and create_web_app.

   Actually, `create_server()` creates bridge/model_server internally as closures. To share them with the web app, we need to either:
   a. Extract the shared objects from create_server, or
   b. Restructure to create shared objects first

   **Approach (b):** In launcher.py, create the components directly:
   - Create `bridge = ReaperOSC(...)` 
   - Create `model_server = ModelServer(settings)` and start it
   - Create `translator = IntentTranslator(model_server)`
   - Create `context_manager = ContextManager(model_server, settings.memory_vault_path)`
   - Create `mcp_server = create_server(settings)` — but this recreates bridge/model_server internally...

   **Simpler approach:** The web app reads status from the same modules (error_log, gpu_monitor, daw_detect are module-level singletons). The bridge status can be checked independently. The web app doesn't need direct access to bridge/model_server objects — it can:
   - Check model server health via HTTP to its /health endpoint (settings.model_api_url)
   - Check DAW connection by trying to send an OSC message
   - Read GPU stats from sysfs

   This is cleaner — the web app is loosely coupled. Use this approach:
   - `create_web_app(settings)` — no bridge/model_server arguments needed
   - Web routes query status via HTTP/sysfs utilities, not direct object access

   **IMPORTANT:** This means updating `web.py` from Plan 04-01. In create_web_app, the bridge/model_server params should be Optional. The home route should use the utility modules (gpu_monitor, daw_detect) for status instead of accessing bridge directly. If bridge is None, show "Not Connected" status.

4. **Start web server** — run uvicorn in a daemon thread:
```python
import uvicorn
from threading import Thread

web_config = uvicorn.Config(app, host=settings.web_host, port=settings.web_port, log_level=settings.log_level)
web_server = uvicorn.Server(web_config)
web_thread = Thread(target=web_server.run, daemon=True)
web_thread.start()
```

5. **Open browser** (if not --no-browser and settings.auto_open_browser):
```python
if not no_browser:
    import threading
    web_url = f"http://{settings.web_host}:{settings.web_port}"
    threading.Timer(2.0, lambda: webbrowser.open(web_url)).start()
```

6. **Start tray icon** (if not --no-tray and settings.tray_enabled):
   - Create TrayIcon with on_quit callback that calls web_server.should_exit = True and stops the MCP server
   - Call `icon.run()` — BLOCKING, main thread

7. **If no tray** (headless mode): 
   - Block on `web_thread.join()` or run MCP server in main thread
   - For stdio MCP: the MCP server runs in main thread via `mcp_server.run(transport="stdio")`
   - For SSE MCP: block on web_thread.join()

**Shutdown:**
- Signal handler for SIGINT/SIGTERM that sets `web_server.should_exit = True` and stops tray
- Tray "Quit" calls the on_quit callback which sets should_exit and stops tray
- Model server stops via its own cleanup

**cli.py** — Update to use launcher:
- Add `--no-browser` flag (default: False)
- Add `--no-tray` flag (default: False)
- In main(): parse args, create Settings, call `launch(settings, transport=args.transport, no_browser=args.no_browser, no_tray=args.no_tray, host=args.host, port=args.port)`
- Keep `--transport` choice: "stdio", "sse", "standalone" (new — standalone = web + tray + SSE MCP)

Actually, simplify: remove SSE from transport choices. New transport options:
- `stdio` — for MCP clients that launch as subprocess (no web, no tray)
- `standalone` — web + tray + MCP SSE (the demo mode)

Default: `standalone`
