Create the context manager module:

```python
class ContextManager:
    """Manages rolling context for the E2B model's translation sessions."""
    
    def __init__(
        self,
        model_server: ModelServer | None = None,
        vault_path: str | Path = "~/.audioshuttle/memory",
        max_messages: int = 40,
        max_chars: int = 6000,
    ):
```

**Methods:**

- `add(role: str, content: str) -> None` — Appends a message to the context list. Each message is `{"role": role, "content": content, "timestamp": iso_format}`. After adding, checks `_should_compact()` and compacts if needed.

- `get_messages() -> list[dict]` — Returns current messages list (for feeding to model server).

- `_should_compact() -> bool` — Returns True if total character count of all messages exceeds `max_chars` OR message count exceeds `max_messages`.

- `_compact() -> None` — Compaction logic:
  1. Split messages: keep last 10 as `recent`, rest as `old`
  2. If `model_server` is available and running, ask it to summarize: send a chat completion request with a summarization prompt. Use `model_server.chat()` with temperature=0.3, max_tokens=300.
  3. Summarization prompt: "Summarize this DAW control session in 200 words or less. Focus on: user preferences, track names referenced, volume/pan settings changed, workflow patterns. Be specific about track names and values."
  4. If model_server is unavailable, do a simple truncation: keep first 100 chars of each old message as a "summary"
  5. Replace messages with: `[{"role": "system", "content": f"Previous session summary: {summary}"}] + recent`
  6. Call `_dump_session(old_messages)` before discarding

- `_dump_session(messages: list[dict]) -> None` — Writes messages to Obsidian vault:
  1. Expand vault_path (default `~/.audioshuttle/memory`)
  2. Create `sessions/` subdirectory if needed
  3. Filename: `{timestamp}.md` where timestamp is ISO format with colons replaced by dashes
  4. Content: Markdown with `# Session {timestamp}` header, then `**User:** content` / **Assistant:** content` for each message
  5. Also create/update `README.md` in vault root with a table of sessions (date, message count, first user message snippet)

- `clear() -> None` — Empties context without dumping.

**Implementation notes:**
- Use `threading.Lock` for thread safety on the messages list
- `model_server.chat()` expects `list[dict]` messages and returns a string response. Check model_server.py for the exact API.
- For the summarization call, construct messages as: `[{"role": "user", "content": summarization_prompt + "\n\n" + concatenated_old_messages}]`
- The vault_path default comes from `Settings.memory_vault_path`

**Module-level convenience:**
- No module-level singleton — ContextManager is created in server.py and passed around

import tempfile, os

# Test with temp vault
