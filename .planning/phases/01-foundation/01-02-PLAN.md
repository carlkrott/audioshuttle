  <name>Task 1: Build ReaperOSC bridge class</name>
  <files>src/audioshuttle/osc_bridge.py</files>
  <action>
    Create `ReaperOSC` class that wraps python-osc for reliable Reaper communication.

    The class must:
    1. **Initialize** with host/port settings from config, create UDP client (send) and UDP server (receive)
    2. **Send commands** via `send_command(address: str, *args)` — wraps python-osc client
    3. **Receive feedback** via a background listener thread on the feedback port
    4. **Track state** internally — maintain a `_state: DAWState` that updates from feedback messages
    5. **Provide high-level methods** that map to Reaper OSC patterns:
       - `transport_play()`, `transport_stop()`, `transport_record()`
       - `set_track_volume(track: int, value: float)` — value 0-1 normalized
       - `set_track_mute(track: int, mute: bool)`
       - `set_track_solo(track: int, solo: bool)`
       - `set_track_pan(track: int, pan: float)` — -1 to 1
       - `select_track(track: int)` — send /track/{n}/select first (THIS MAY FIX THE TRACK COMMAND BUG)
       - `get_track_count()` — probe tracks by sending /track/{n}/name until no response
       - `refresh_state()` — request current state by probing all tracks

    **Critical fix for track commands:** Before sending any track-specific command (mute/solo/volume/pan), FIRST send `/track/{n}/select 1` to select the track in Reaper. Reaper's OSC may require track selection before accepting per-track commands. The Default.ReaperOSC file shows patterns like `b/track/mute b/track/@/mute` where `@` means "currently selected track". Try BOTH approaches:
    - Explicit addressing: `/track/1/mute 1`
    - Select-then-command: `/track/1/select 1` then `/track/mute 1`

    **Feedback listener:** Start a ThreadingOSCUDPServer on the feedback port (9000) with a dispatcher that:
    - Logs all received messages for debugging
    - Updates internal state (track names, volumes, transport position)
    - Stores the last N messages in a deque for inspection

    **Connection handling:**
    - `is_connected` property — True if Reaper has sent feedback in the last 5 seconds
    - `_ping()` — send `/play` with current state to trigger a feedback response
    - Auto-reconnect: if no feedback for 10s, log warning

    Import from audioshuttle.models: TrackState, TransportState, DAWState, CommandResult
    Import from audioshuttle.config: Settings (or accept host/port in __init__)
  </action>
