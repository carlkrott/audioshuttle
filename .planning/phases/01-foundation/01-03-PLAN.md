  <name>Task 1: Create MCP server with DAW control tools</name>
  <files>src/audioshuttle/server.py, src/audioshuttle/cli.py</files>
  <action>
    **server.py** — Create an MCP server using `fastmcp` (not `mcp` — fastmcp has a simpler API):

    ```python
    from fastmcp import FastMCP
    mcp = FastMCP("AudioShuttle", instructions="DAW control server for Reaper")
    ```

    Create a factory function `create_server(settings: Settings) -> FastMCP` that:
    1. Instantiates a `ReaperOSC` bridge with settings
    2. Registers the following MCP tools:

    **Tools to register:**

    - `list_tracks()` → returns JSON list of track names, numbers, current state
      - Calls `bridge.refresh_state()` then returns `bridge._state.tracks`
      - Returns: `{"tracks": [{"number": 1, "name": "Drums", "volume": 0.75, ...}]}`

    - `get_transport()` → returns current transport state
      - Returns: `{"playing": false, "position_seconds": 0.0, "tempo": 120.0}`

    - `transport_control(action: str)` → play, stop, record, pause
      - Validates action is one of: play, stop, record, pause
      - Calls appropriate bridge method
      - Returns: `{"success": true, "action": "play"}`

    - `set_track_volume(track: int, volume: float)` → set volume (0-1, clamped)
      - Clamps volume to [0.0, 1.0]
      - Calls `bridge.set_track_volume(track, volume)`
      - Returns: `{"success": true, "track": 1, "volume": 0.75}`

    - `set_track_mute(track: int, mute: bool)` → mute/unmute
      - Calls `bridge.set_track_mute(track, mute)`
      - Returns: `{"success": true, "track": 1, "muted": true}`

    - `set_track_solo(track: int, solo: bool)` → solo/unsolo
      - Returns: `{"success": true, "track": 1, "soloed": true}`

    - `set_track_pan(track: int, pan: float)` → set pan (-1 to 1, clamped)
      - Clamps pan to [-1.0, 1.0]
      - Returns: `{"success": true, "track": 1, "pan": -0.5}`

    - `get_daw_state()` → full DAW state snapshot
      - Returns: full DAWState as JSON (tracks + transport + project name)

    Each tool must:
    - Have a clear docstring (this becomes the tool description for AI)
    - Include parameter descriptions in the function signature
    - Return dict/JSON (not Pydantic models — MCP tools should return serializable data)
    - Handle errors gracefully (bridge not connected, Reaper not responding)

    **cli.py** — Simple CLI entry point:
    ```python
    import argparse
    import asyncio
    from audioshuttle.config import Settings
    from audioshuttle.server import create_server

    def main():
        parser = argparse.ArgumentParser(description="AudioShuttle MCP Server")
        parser.add_argument("--transport", choices=["stdio", "sse"], default="stdio")
        parser.add_argument("--host", default="127.0.0.1")
        parser.add_argument("--port", type=int, default=8765)
        args = parser.parse_args()

        settings = Settings()
        server = create_server(settings)

        if args.transport == "stdio":
            server.run(transport="stdio")
        else:
            server.run(transport="sse", host=args.host, port=args.port)

    if __name__ == "__main__":
        main()
    ```

    Use the approach for track commands that was diagnosed as working in Plan 01-02 (check SUMMARY for which approach works — select-first vs direct).
