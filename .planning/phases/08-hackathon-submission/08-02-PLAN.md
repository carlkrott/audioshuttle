Create a Dockerfile for the E4B model server based on llama.cpp at `/home/korphaus/audioshuttle/Dockerfile.e4b`.

Since we cannot redistribute the Gemma model weights, this Dockerfile should:
1. Use `ubuntu:22.04` as base (ROCm compatibility)
2. Install build deps (cmake, build-essential, ROCm or CPU backend)
3. Clone and build `llama.cpp` from source
4. Expose port 8102
5. Set up a volume mount point `/models` for GGUF files
6. Entrypoint: `llama-server` with args configured via env vars

Important considerations:
- Build with ROCm support IF ROCm is detected, otherwise CPU-only (use CMake flags)
- The model GGUF files are NOT included in the image — they're too large. User must download separately.
- Document the expected model filenames (gemma-4-E4B-it-UD-Q4_K_XL.gguf + mmproj)

Launch configuration via env vars:
- `MODEL_PATH` — path to GGUF (default: `/models/gemma-4-E4B-it-UD-Q4_K_XL.gguf`)
- `MMPROJ_PATH` — path to mmproj (default: `/models/gemma-4-e4b-mmproj-BF16.gguf`)
- `PORT` — server port (default: 8102)
- `CONTEXT_SIZE` — context window size (default: 81920)
- `N_GPU_LAYERS` — GPU layers for acceleration (default: 0 for CPU-only Docker)
- `HOST` — bind address (default: 0.0.0.0)

Add a `--help` note in comments about how to download the model files.
Suffix the filename as `.e4b` not `.e4b.Dockerfile` so Docker recognizes it.

<task type="auto">
<name>Create docker-compose.yml and .env.example</name>
<files>docker-compose.yml, .env.example</files>
<action>
Create `/home/korphaus/audioshuttle/docker-compose.yml` orchestrating two services:

```yaml
services:
  e4b:
    build:
      context: .
      dockerfile: Dockerfile.e4b
    ports:
      - "8102:8102"
    volumes:
      - ./models:/models  # Mount GGUF files here
    environment:
      - MODEL_PATH=/models/gemma-4-E4B-it-UD-Q4_K_XL.gguf
      - MMPROJ_PATH=/models/gemma-4-e4b-mmproj-BF16.gguf
      - PORT=8102
      - CONTEXT_SIZE=81920
      - N_GPU_LAYERS=0
      - HOST=0.0.0.0
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8102/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  audioshuttle:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8765:8765"
    depends_on:
      e4b:
        condition: service_healthy
    environment:
      - AUDIOSHUTTLE_MODEL_API_URL=http://e4b:8102/v1/chat/completions
      - AUDIOSHUTTLE_TRANSPORT=stdio
      - AUDIOSHUTTLE_HOST=0.0.0.0
      - AUDIOSHUTTLE_PORT=8765
    env_file: .env
    stdin_open: true   # For MCP stdio transport
    tty: true
```
