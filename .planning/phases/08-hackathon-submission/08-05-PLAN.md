<task type="auto">
<name>Configure git remote and push codebase</name>
<files>N/A</files>
<action>
Configure the local repository's remote and push the cleaned codebase.

1. Check current remotes: `git remote -v`
2. If no remote exists, add the GitHub remote:
   ```
   git remote add origin git@github.com:korphaus/audioshuttle.git
   ```
   (Use HTTPS fallback: `https://github.com/korphaus/audioshuttle.git` if SSH key isn't configured.)

3. Ensure the current branch is named `main` (not `master`):
   ```
   git branch -m main 2>/dev/null; git symbolic-ref HEAD refs/heads/main 2>/dev/null
   ```

4. Stage all files (code only — .planning is now gitignored):
   ```
   git add .
   ```

5. Show what would be committed: `git status` — verify no personal info, .planning files, model files, or secrets are staged.

6. Commit if there are uncommitted changes:
   ```
   git commit -m "feat: initial hackathon submission — AudioShuttle v0.1.0"
   ```

7. Push to GitHub:
   ```
   git push -u origin main
   ```

If git push fails due to authentication:
- Try the gh CLI: `gh repo push`
- Or set up a GitHub token: `git remote set-url origin https://<token>@github.com/korphaus/audioshuttle.git`
- Or suggest the user run `gh auth login` first
</action>
<verify>git remote -v && git log --oneline -1 && git status --short | head -5</verify>
<done>GitHub remote configured. Clean codebase pushed to `main` branch. No personal files or .planning in the push.</done>
</task>
