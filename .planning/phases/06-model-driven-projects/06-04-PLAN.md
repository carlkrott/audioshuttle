**Fix 1: Increase FX timeout** — Change `wait=4.0` to `wait=10.0` at the `_fx_trigger("add", ...)` call in Step 7 (around line 1550).

**Fix 2: Add liveness gate** — Before each FX trigger in Step 7, check `_watcher_alive()`. If dead, wait up to 15s for recovery:

```python
for fx_def in fx_chain:
    plugin_name = fx_def.get("name", "")
    if not plugin_name:
        continue
    
    # Wait for watcher if it's unresponsive
    if not _watcher_alive():
        logger.info("create_genre_project: waiting for watcher before FX...")
        for _ in range(5):
            _time.sleep(3.0)
            if _watcher_alive():
                break
        else:
            logger.warning("create_genre_project: watcher still dead, skipping remaining FX")
            break
    
    result = self._fx_trigger("add", track_idx, plugin_name, wait=10.0)
    # ... rest of FX handling
```

**Fix 3: Reduce state dump frequency** — In Steps 5 and 6, the pipeline calls `refresh_state()` multiple times. Remove unnecessary `refresh_state()` calls between steps where the state is predictable. Keep only the verification calls in `_verify_project_state()`.
