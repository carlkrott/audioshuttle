The Lua watcher dies during multi-instrument `generate_project()` calls (observed: drums works, bass fails). Root cause: blocking operations like `InsertMedia` prevent the defer loop from firing heartbeats, and the 5s threshold in `_watcher_alive()` is too tight.

2. Add a retry mechanism: if heartbeat file is stale, wait 3s and check again before declaring dead
3. Also check tick counter monotonic increase (compare current tick to last known tick) as secondary liveness check — if tick is still incrementing, watcher is alive even if mtime is old

**Changes to `generate_project()` loop (around line 1002):**

5. After MIDI import, wait for trigger consumption with longer timeout (increase from 5s to 8s via range(40) at 0.2s intervals)

```python
# Updated _watcher_alive()
def _watcher_alive(self, retry: bool = True) -> bool:
    import time
    watcher_file = "/tmp/audioshuttle_watcher_alive"
    for attempt in range(3 if retry else 1):
        try:
            mtime = os.path.getmtime(watcher_file)
            age = time.time() - mtime
            if age < 15.0:
                # Secondary check: read tick counter
                try:
                    with open(watcher_file) as f:
                        tick = int(f.read().strip() or "0")
                    last = getattr(self, "_last_watcher_tick", -1)
                    self._last_watcher_tick = tick
                    if tick > last or last < 0:
                        return True
                except (ValueError, OSError):
                    return True  # mtime check alone is sufficient
        except OSError:
            pass
        time.sleep(3.0)  # wait before retry
    return False
```
