Create `src/audioshuttle/voice.py`:

1. **VoicePipeline class:**
   - `__init__(self, stt_engine, model_server=None, bridge=None, translator=None)`
   - `async def process_audio(self, audio_bytes: bytes, filename: str, cleanup: bool = True) -> dict`
     - Saves audio to temp file (tempfile.NamedTemporaryFile)
     - Calls STTEngine.transcribe() on temp file
     - If cleanup=True AND model_server available: calls `_format_text()` on raw transcription
     - If cleanup=True AND model_server NOT available: returns error "Model server required for voice formatting. The model is integral to this system."
     - If cleanup=False: skips formatting, uses raw text
     - Calls translator.translate() on the (optionally formatted) text
     - Executes translated command via bridge
     - Returns: `{"transcription": str, "formatted": str|None, "command": dict, "success": bool, "error": str|None}`
     - Cleans up temp file

   - `async def _format_text(self, raw_text: str) -> str`
     - Sends raw text to model_server.chat() with formatting prompt:
       ```
       Clean up this voice transcription for DAW command processing. Remove filler words (um, uh, like), 
       fix false starts, normalize to clear instruction language. Output ONLY the cleaned text, nothing else.
       
       Raw: "{raw_text}"
       ```
     - Returns cleaned text
     - This is LIGHT NORMALIZATION only — does not convert to OSC or interpret the command

   - `process_text_only(self, text: str, cleanup: bool = True) -> dict`
     - Same as process_audio but skips STT, for direct text input
     - Useful for testing the pipeline without audio

Create `src/audioshuttle/hotkey.py`:

1. **VoiceHotkey class** — system-wide Alt+Space recording:
   - `__init__(self, voice_pipeline, callback=None)`
   - `start()` — registers hotkey and starts listening
   - `stop()` — unregisters and cleans up
   - On Alt+Space press: starts recording audio from default mic
   - On Alt+Space release: stops recording, sends audio to voice_pipeline.process_audio()
   - Callback receives the result dict

2. **Platform-specific implementation:**
   - **Primary (KDE Wayland):** Use `xdg-desktop-portal` GlobalShortcuts D-Bus interface
     - Register "audioshuttle-voice" shortcut via org.freedesktop.portal.GlobalShortcuts
     - Request permission via D-Bus dialog on first use
   - **Fallback (X11):** Use `python-xlib` or `keyboard` library for global key capture
   - **Audio capture:** Use `sounddevice` or `pyaudio` to record from default mic while key is held
   - **If hotkey registration fails:** Log warning, fall back to web-only voice recording

3. **Config additions** (in config.py):
   - `voice_hotkey: str = "alt+space"` — configurable hotkey combo
   - `voice_cleanup: bool = True` — default for cleanup toggle
   - `voice_sample_rate: int = 16000` — recording sample rate (Whisper prefers 16kHz)

4. **Integration with launcher:** VoiceHotkey should be started in launcher.py after all components are initialized

Create `tests/test_voice.py`:

- Test _format_text() with mocked model_server.chat()
- Test cleanup=False skips formatting

- Use `wave` module to create tiny test WAV in test fixtures

Do NOT test the global hotkey directly (requires display server) — test the VoicePipeline independently.
