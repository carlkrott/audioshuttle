Improve `src/audioshuttle/launcher.py`:

1. **Add `--no-model` CLI flag** (in cli.py, passed to launch()):
   - Skips model server start entirely
   - Useful for testing without GPU or when model is not needed

2. **Component isolation** — each component starts independently:
   - Model server fails → continue without model (text-only + no voice formatting)
   - Voice hotkey fails → continue with web-only voice recording
   - Tray fails → continue without tray icon
   - Only Reaper OSC bridge and web server are required; everything else is optional
   - Log which components started successfully at INFO level

3. **Integrate VoiceHotkey in startup sequence:**
   - After all components initialized, create VoicePipeline with STT + model + translator + bridge
   - Create VoiceHotkey with pipeline
   - Start hotkey listener in a background thread
   - If hotkey registration fails (Wayland issues): log warning, continue without global hotkey

4. **Browser auto-open logic:**
   - Standalone mode (default): auto-open browser unless `--no-browser`
   - stdio mode: never open browser (already fixed in Plan 06)

5. **Graceful shutdown:**
   - SIGINT/SIGTERM handler stops components in order:
     1. VoiceHotkey (stop recording/listening)
     2. Web server (uvicorn)
     3. Model server (kill subprocess)
     4. Tray icon (remove)
     5. OSC bridge (close sockets)
