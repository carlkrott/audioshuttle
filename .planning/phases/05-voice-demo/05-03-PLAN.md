Create `src/audioshuttle/midi_generator.py`:

1. **MIDIGenerator class:**
   - `generate(role: str, bars: int = 16, seed: int | None = None) -> dict`
     - role: "drums", "rhythm", "lead", "melody"
     - Returns: `{"role": str, "bars": int, "pattern": list[list[int]], "seed": int}`
     - pattern is bars × steps_per_bar (4 for drums/rhythm, 8 for lead/melody)
     - Each cell is velocity 0-127 (0 = off, 1-127 = note velocity)

   - **Role-specific generation rules:**
     - **Drums:** Kick on beats 1,3; snare on 2,4; hi-hat every 8th with random velocity variation (±20). 4 steps per bar (quarter notes).
     - **Rhythm:** Repetitive 2-4 bar cycle, chord tones, higher density. 8 steps per bar (8th notes).
     - **Lead:** More melodic, wider intervals, ~50% density, occasional rests. 8 steps per bar.
     - **Melody:** Scale-constrained (C major default), stepwise motion with occasional leaps (20% chance), ~60% density. 8 steps per bar.

   - Use Python `random` module with seed for reproducibility

2. **JSON serialization:** Pattern data is JSON-serializable for passing to the template and to E2B.

Create `src/audioshuttle/web_routes/midi_tab.py`:

1. **APIRouter** with prefix=""
2. **GET `/midi`** — renders midi.html with:
   - Current pattern (empty grid on first load)
   - Role selector defaulting to "drums"
   - Seed value (random or user-specified)
3. **POST `/midi/generate`** — accepts form with `role` and optional `seed`:
   - Calls MIDIGenerator.generate()
   - Stores pattern in `app.state.midi_pattern` (or session)
   - Redirects back to /midi with pattern displayed
4. **POST `/midi/send`** — accepts form with `description` text:
   - Gets current pattern from app.state
   - Constructs prompt for E2B: "Insert a {role} MIDI pattern into the DAW. User says: {description}. Pattern data: {json}"
   - Calls translator or model_server directly to get formatted command
   - Executes the command via bridge
   - Redirects back to /midi with result message

Create `src/audioshuttle/templates/midi.html`:

1. **Role selector dropdown:** Drums, Rhythm, Lead, Melody
2. **Seed input** (optional, blank = random)
3. **"Randomize" button** — POST to /midi/generate
4. **16-bar step sequencer grid:**
   - CSS grid layout: rows = 4 (or 8) steps, columns = 16 bars
   - Each cell is a small square, colored by velocity (0=dark gray, 1-64=dim color, 65-127=bright color)
   - Active cells show the velocity as background color intensity
   - Use inline CSS (match dark theme)
5. **"Send to Track" section below the grid:**
   - Label: "Where do you want this? Describe the instrument, key, and tempo to add it."
   - Text input (multiline, 2-3 rows)
   - "Send" button — POST to /midi/send
6. **Result message area** (shows after send: success or error)
