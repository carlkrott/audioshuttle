**1. Header:**
- Project name "AudioShuttle" with tagline: "AI-agnostic DAW control via MCP — bridge any LLM to Reaper for accessible audio production"
- Badges: Python 3.12+, License MIT, Tests passing
- One-paragraph elevator pitch focused on accessibility

**2. 🎯 The Problem:**
- Professional audio production requires complex mouse/keyboard workflows
- Musicians with motor disabilities, RSI, or visual impairments are locked out
- Existing solutions are vendor-locked and expensive

**3. 💡 The Solution:**
- MCP (Model Context Protocol) server that bridges ANY LLM to Reaper DAW
- Speak or type natural language commands: "mute the drums", "turn up the vocals"
- Two-brain architecture: External AI (any) = creative producer, Embedded Gemma E2B = precise audio engineer
- Works with Gemini, Claude, GPT, or any MCP-compatible client

**4. 🏗 Architecture:**
- ASCII art diagram showing: User → LLM Client → (MCP) → AudioShuttle Server → (OSC) → Reaper
- Brief explanation of each component
- Mention the 20 MCP tools available

**5. 🚀 Quick Start:**
```bash
# Install
git clone https://github.com/user/audioshuttle
cd audioshuttle
python -m venv .venv && source .venv/bin/activate
pip install -e ".[all]"

# Start Reaper (must be running)
reaper &

# Launch AudioShuttle
audioshuttle

# Or for MCP client integration:
audioshuttle --transport stdio
```

**6. 🛠 Configuration:**
- Reference `~/.audioshuttle/config.json` auto-created on first run
- Key settings: reaper_host/port, model settings, web_port
- DAW type selection (Reaper/Ardour)
